# SE object

Pivots the raw data into a matrix and constructs a formal
SummarizedExperiment object, linking the quantification data perfectly
to the row (peptide) and column (sample) metadata. Crucially, this step
mathematically sanitizes the data by replacing the artificial software
imputation values (1.01) with true R NAs to prevent statistical
artifacts.

``` r
quant_wide <- df.main %>%
  select(EG.PrecursorId, R.FileName, FG.Quantity) %>%
  pivot_wider(
    names_from = R.FileName,
    values_from = FG.Quantity,
    values_fn = max # Resolves any rare duplicate precursor entries in a single run
  ) %>%
  as.data.frame()

rownames(quant_wide) <- quant_wide$EG.PrecursorId
quant_matrix <- as.matrix(quant_wide[, -1]) 


col_data <- df.main %>%
  select(R.FileName, SampleID, R.Condition, Rep, group) %>%
  distinct() %>%
  as.data.frame()

rownames(col_data) <- col_data$R.FileName

quant_matrix <- quant_matrix[, rownames(col_data)] # *** important ***


row_data <- df.main %>%
  select(
    EG.PrecursorId,
    Sequence = PEP.StrippedSequence,
    Proteins = PG.ProteinGroups,
    PEP = EG.PEP,

    Potential.contaminant,
    IsDecoy = EG.IsDecoy
```

# QF object

Upgrades our data to a QFeatures structure for advanced proteomics
processing. We perform rigorous quality filtering here: dropping
peptides missing in \>70% of samples, removing decoys/contaminants,
retaining strictly high-confidence peptides (PEP \< 0.05), and finally
applying a log2 transformation to establish a normal distribution for
downstream algorithms.
